# [高精度算法](https://www.modb.pro/db/158430)

## 01 [高精度加法](https://www.modb.pro/db/158430)
高精度加法是比较基础的高精度算法，本文以下的的例子会使用高精度对514+495这个算式进行运算。

要使用高精度的话，首先需要要运用数组对两个加数进行储存。在处理前，高精度数通常会以字符串的形式储存。我们需要通过数组，将每一位的数字分别储存在数组的每一个元素，用n位的数组来记录一个n位数字，如下面表格。

| 数组   | 第4位 | 第三位 | 第二位 | 第一位 |
|--------|-------|--------|--------|--------|
| a      |       | 5      | 1      | 4      |
| b      |       | 4      | 9      | 5      |

在解决了储存问题之后，我们就可以开始运算了。高精度加法中的运算思路类似于竖式加法，对每一位进行加法和进位。对于是10进制而言，某一位上的数达到10就会向前进一步，同时保留那一位的个位数。

| 数组     | 第4位 | 第三位 | 第二位 | 第一位 |
|----------|-------|--------|--------|--------|
| a        |       | 5      | 1      | 4      |
| b        |       | 4      | 9      | 5      |
| 中间产物 |       | 9      | 10     | 9      |
| 预处理1  |       | 10     | 0      | 9      |
| 预处理2  | 1     | 0      | 0      | 9      |
| 结果     | 1     | 0      | 0      | 9      |

从上面的过程我们很容易得到一个进位的规律，必须要从低位向高位进位，否则可能会发生顺序上的错误。同时也要注意到加法可能会改变结果的最大位数，把最后的结果输出，也就完成了最基础的高精度加法运算。

## 02 [高精度乘法](https://www.modb.pro/db/158428)
高精度乘法实质上是在高精度加法上更进一步，具体原理相似。我们以514*495为例。

与高精度加法类似，在运用高精度乘法时，同样也需要把数字用数组的形式进行储存，这里不再过多强调。

| 数         | 第6位 | 第5位 | 第4位 | 第3位 | 第2位 | 第1位 |
|------------|-------|-------|-------|-------|-------|-------|
| a          |       |       |       | 5     | 1     | 4     |
| b          |       |       |       | 4     | 9     | 5     |
| a*b[1]     |       |       |       | 25    | 5     | 20    |
| a*b[2]     |       |       | 45    | 9     | 36    |       |
| a*b[3]     |       | 20    | 4     | 16    |       |       |
| 中间产物   |       | 20    | 49    | 50    | 41    | 20    |
| 进位结果   | 2     | 5     | 4     | 4     | 3     | 0     |
| 结果       | 2     | 5     | 4     | 4     | 3     | 0     |

仔细观察这个表格，不难看出，a[i]*b[j]的结果全在i+j-1的位置上。利用这个特性，我们可以很方便计算出中间产物的位置，最后在对整个算式进行进位。同高精度加一样，在进位中同样要注重进位的方向。

## 例题：高精度加法
# P1601 A+B Problem（高精）

## 题目描述

高精度加法，相当于 a+b problem，**不用考虑负数**。

## 输入格式

分两行输入。$a,b \leq 10^{500}$。

## 输出格式

输出只有一行，代表 $a+b$ 的值。

## 输入输出样例 #1

### 输入 #1

```
1
1
```

### 输出 #1

```
2
```

## 输入输出样例 #2

### 输入 #2

```
1001
9099
```

### 输出 #2

```
10100
```

## 说明/提示

$20\%$ 的测试数据，$0\le a,b \le10^9$；

$40\%$ 的测试数据，$0\le a,b \le10^{18}$。

```c
#include <stdio.h>
#include <string.h>
int main()
{
    char num1[20];
    char num2[20];
    int num_1[20] = {0};
    int num_2[20] = {0};
    int num_[40]={0};
    char num[40];
    int i, t=0;
    int len1, len2;
    scanf("%s%s", num1, num2);
    len1 = strlen(num1);
    len2 = strlen(num2);
    for(i=0; i<len1; i++){
        num_1[19-i] = num1[len1-1-i] - '0';
    }
    for(i=0; i<len2; i++){
        num_2[19-i] = num2[len2-1-i] - '0';
    }
    for(i=0; i<20; i++){
        num_[39-i] = (num_1[19-i] + num_2[19-i])%10 + t;
        t = (num_1[19-i] + num_2[19-i])/10;
    }
    if(t != 0) num_[20] = t;
    for(i=0; i<40; i++){
        if(num_[i] != 0){
            break;
        }
    }
    for(t=0; t<40-i; t++){
        num[t] = num_[t+i] + '0';
    }
    num[40-i] = '\0';
    printf("%s", num);

    return 0;
}

```
