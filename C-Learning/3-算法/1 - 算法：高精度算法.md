# [高精度算法](https://www.modb.pro/db/158430)

## 01 [高精度加法](https://www.modb.pro/db/158430)
高精度加法是比较基础的高精度算法，本文以下的的例子会使用高精度对514+495这个算式进行运算。

要使用高精度的话，首先需要要运用数组对两个加数进行储存。在处理前，高精度数通常会以字符串的形式储存。我们需要通过数组，将每一位的数字分别储存在数组的每一个元素，用n位的数组来记录一个n位数字，如下面表格。

| 数组   | 第4位 | 第三位 | 第二位 | 第一位 |
|--------|-------|--------|--------|--------|
| a      |       | 5      | 1      | 4      |
| b      |       | 4      | 9      | 5      |

在解决了储存问题之后，我们就可以开始运算了。高精度加法中的运算思路类似于竖式加法，对每一位进行加法和进位。对于是10进制而言，某一位上的数达到10就会向前进一步，同时保留那一位的个位数。

| 数组     | 第4位 | 第三位 | 第二位 | 第一位 |
|----------|-------|--------|--------|--------|
| a        |       | 5      | 1      | 4      |
| b        |       | 4      | 9      | 5      |
| 中间产物 |       | 9      | 10     | 9      |
| 预处理1  |       | 10     | 0      | 9      |
| 预处理2  | 1     | 0      | 0      | 9      |
| 结果     | 1     | 0      | 0      | 9      |

从上面的过程我们很容易得到一个进位的规律，必须要从低位向高位进位，否则可能会发生顺序上的错误。同时也要注意到加法可能会改变结果的最大位数，把最后的结果输出，也就完成了最基础的高精度加法运算。

## 02 [高精度乘法](https://www.modb.pro/db/158428)
高精度乘法实质上是在高精度加法上更进一步，具体原理相似。我们以514*495为例。

与高精度加法类似，在运用高精度乘法时，同样也需要把数字用数组的形式进行储存，这里不再过多强调。

| 数         | 第6位 | 第5位 | 第4位 | 第3位 | 第2位 | 第1位 |
|------------|-------|-------|-------|-------|-------|-------|
| a          |       |       |       | 5     | 1     | 4     |
| b          |       |       |       | 4     | 9     | 5     |
| a*b[1]     |       |       |       | 25    | 5     | 20    |
| a*b[2]     |       |       | 45    | 9     | 36    |       |
| a*b[3]     |       | 20    | 4     | 16    |       |       |
| 中间产物   |       | 20    | 49    | 50    | 41    | 20    |
| 进位结果   | 2     | 5     | 4     | 4     | 3     | 0     |
| 结果       | 2     | 5     | 4     | 4     | 3     | 0     |

仔细观察这个表格，不难看出，a[i]*b[j]的结果全在i+j-1的位置上。利用这个特性，我们可以很方便计算出中间产物的位置，最后在对整个算式进行进位。同高精度加一样，在进位中同样要注重进位的方向。

# 例题

## 例题：高精度加法

### 题目描述

高精度加法，相当于 a+b problem，**不用考虑负数**。

### 输入格式

分两行输入。$a,b \leq 10^{500}$。

### 输出格式

输出只有一行，代表 $a+b$ 的值。

### 输入输出样例 #1

### 输入 #1

```
1
1
```

### 输出 #1

```
2
```

### 输入输出样例 #2

### 输入 #2

```
1001
9099
```

### 输出 #2

```
10100
```
### 代码示例
```c
#include <stdio.h>
#include <string.h>
int main()
{
    int a[1001]={0}, b[1001]={0}, c[1002]={0};  
    char A[1001], B[1001];
    char result[1002];
    int len1, len2, len;
    int i;

    scanf("%s%s", A, B);
    len1 = strlen(A);
    len2 = strlen(B);
    len = (len1 > len2) ? len1 : len2;
    // 逆序存储。若正序存储，由于len1与len2不一定相等，个位很难对齐。
    // 存储时，用1起头，便于表示第1位。此时，精度最大为1000位。
    for(i=1; i<=len1; i++){
        a[i] = A[len1 - i] - '0';
    }
    for(i=1; i<=len2; i++){
        b[i] = B[len2 - i] - '0';
    }
    // 开始相加操作，此步注意，结果数组必须初始化为0。
    for(i=1; i<=len; i++){
        c[i] += a[i] + b[i];
        c[i+1] = c[i] / 10;
        c[i] %= 10;
    }
    // 若有进位，长度加一。
    if(c[len+1] > 0) len++;

    for(i=0; i<len; i++){
        result[i] = c[len-i] + '0';
    }

    result[len] = '\0';

    printf("%s", result);

    return 0;
}

```

#include <stdio.h>
#include <string.h>
int main()
{
    int a[1000]={0}, b[1000]={0}, c[1001]={0};
    char A[1001], B[1001];
    char result[1002];
    int len1, len2, len;
    int i;

    scanf("%s%s", A, B);
    len1 = strlen(A);
    len2 = strlen(B);
    len = (len1 > len2) ? len1 : len2;
    
    for(i=1; i<=len1; i++){
        a[i] = A[len1 - i] - '0';
    }
    for(i=1; i<=len2; i++){
        b[i] = B[len2 - i] - '0';
    }
    // 与加法的区别：
    for(i=1; i<=len; i++){
        c[i] += a[i] - b[i];
        if(c[i] < 0) c[i+1] += -1; 
        c[i] += 10;
        c[i] %= 10;
    }
	for(i=len; i>=1; i--){
		if(c[i] != 0){
			len = i;  // 找到不是0的位置。此时结果的长度就是 i 。
			break;
		}
	}
	
    for(i=0; i<len; i++){
        result[i] = c[len-i] + '0';
    }

    result[len] = '\0';

    printf("%s", result);

    return 0;
}

## 例题：高精度乘法

### 题目背景

高精度乘法模板题。

### 题目描述

给出两个非负整数，求它们的乘积。

### 输入格式

输入共两行，每行一个非负整数。

### 输出格式

输出一个非负整数表示乘积。

### 输入输出样例 #1

#### 输入 #1

```
1 
2
```

#### 输出 #1

```
2
```
### 代码示例
```c
#include <stdio.h>
#include <string.h>
int main()
{
    int a[1001]={0}, b[1001]={0}, c[2001]={0};  
    char A[1001], B[1001];
    char result[2001];
    int len1, len2, len;
    int i, j;

    scanf("%s%s", A, B);
    len1 = strlen(A);
    len2 = strlen(B);
    len = (len1 > len2) ? len1 : len2;
    // 逆序存储。若正序存储，由于len1与len2不一定相等，个位很难对齐。
    // 存储时，用1起头，便于表示第1位。此时，精度最大为1000位。
    for(i=1; i<=len1; i++){
        a[i] = A[len1 - i] - '0';
    }
    for(i=1; i<=len2; i++){
        b[i] = B[len2 - i] - '0';
    }

    // a[i]*b[j]的结果全在i+j-1的位置上。利用这个特性，可以计算出中间产物的位置。
    for(i=1; i<=len1; i++){
        for(j=1; j<=len2; j++){
            c[i+j-1] += a[i]*b[j];
        }
    }

    // 寻找c目前的实际长度
    for(i=2000; i>=1; i--){
        if(c[i] != 0){
            len = i;
            break;
        }
    }

    // 接下来进行进位。
    for(i=1; i<=len; i++){
        c[i+1] += c[i] / 10;
        c[i] %= 10;
    }
    // 若有进位，长度加一。
    if(c[len+1] > 0) len++;

    for(i=0; i<len; i++){
        result[i] = c[len-i] + '0';
    }

    result[len] = '\0';

    printf("%s", result);

    return 0;
}
```
