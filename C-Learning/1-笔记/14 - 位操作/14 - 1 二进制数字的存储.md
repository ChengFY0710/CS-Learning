# 14 - 1 二进制数字的存储

## 01 计算机的存储单位
位 bit   字节 Byte（B）
**通常，一个字节表示八位**，C语言用**字节**表示储存系统字符集所需的大小，所以C字节可能是8位、9位、16位或其他值。不过，描述**存储器芯片和数据传输率**中所用的字节指的是**8位字节**。
> [!Note]
> 计算机界通常用“八位组”来特指八位字节。

## 02 二进制整数
![14 - 1 - 1](/引用文件/14%20-%201%20-%201.png)
**位编号**：在一字节中，从左至右依次标有“7 6 5 ...”，称为位编号。
**高阶位**：位编号为 7 的一位称为高阶位。
**低阶位**：位编号为 0 的一位称为低阶位。
**位组合**：一个二进制数的8位数字。
### （1）无符号整数 unsigned
共八位，可以表示 0 ~ 255 范围内的整数。
### （2）有符号整数 signed
####  1 符号量表示法
用高阶位储存符号，高阶位是0，为正数，高阶位为1，为负数。
用这种方法储存二进制数字，可以储存-127 ~ +127 范围内的数字。
>[!caution]
>用这种方法，10000000和00000000都表示0，造成了浪费。
#### 2 二进制补码
对于正数而言，高阶位为0，后七位表示数值，因此能表示0~127范围的正数。
对于负数而言，高阶位为1，后七位的数字正常。从一个9位形式的数字（100000000）减去一个负数的位组合，结果是该负数的量。
>[!tip]
>例如，一个位组合10000000，作为一个无符号整数，它表示128，但作为一个用二进制补码表示的有符号整数，它表示的值为 100000000（9位256）- 10000000 = 10000000，结果为128，则该位组合（10000000）表示 -128。

>[!tip]
>用正数可以将其转换为负数的二进制补码。
>![14 - 1 - 2](/引用文件/14%20-%201%20-%202.png)

#### 3 二进制反码
通过**反转位组合中的每一位**形成一个负数，能表示-127~127内的整数，同样也存在00000000和11111111两个0的浪费。

## 03 二进制浮点数
浮点数分两部分储存：**二进制小数**和**二进制指数**。一般说来，小数部分是由几个数字**乘以2的指定次幂**组成。
>[!note]
>二进制小数 .101 即 1/2 + 0/4 + 1/8 = 0.625

## 04 十进制转化为二进制
### （1）整数部分转换
```c
// 25 ==> 
 25 % 2 ==> 1
 25 / 2 ==> 12
 12 % 2 ==> 0
 12 / 2 ==> 6
 6 % 2 ==> 0
 6 / 2 ==> 3
 3 % 2 ==> 1
 3 / 2 ==> 1
 1 % 2 ==> 1
 1 / 2 ==> 0 end.
 result: 11001
 ```
 与十进制逻辑上一致。
 ### （2）小数部分转换
 ```c
 // 0.25 ==>
 0.25 * 2 ==> 0.5 ==> 0
 0.5 - 0 ==> 0.5
 0.5 * 2 ==> 1.0 ==> 1
 1.0 - 1 ==> 0.0 end
 result: 0.01
 ```
 每次把小数乘2转至个位初，取值，减去，重复，直到为0。
