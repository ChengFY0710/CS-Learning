# 15 - 1 文件的性质

## 01 文件是什么
文件(file)通常是在磁盘或固态硬盘上的一段已命名的存储区。C把文件看作是**一系列连续的字节**，**每个字节都能被单独读取**。由于其他环境中可能无法完全对应这个模型，C提供了两种文件模式：**文本模式**和**二进制模式**。

## 02 文本模式和二进制模式
为规范文本文件的处理，C提供两种访问文件的途径：**二进制模式**和**文本模式**。

+ 在二进制模式中，程序可以访问文件的每个字节。
+ 在文本模式中，程序所见的内容和文件的实际内容不同。

## 03 文件结尾 EOF
EOF是C中标记文件结尾的特殊值，通常是-1。只有C程序在**读到超过文件末尾**时才会发现文件的结尾。

## 04 文件指针
```c
int main()
{
    FILE * fp;
    fp = fopen(text, "r");
}
```
FILE 是定义在**stdio.h**中的**派生类型**。fp并不指向实际的文件，它指向一个包含文件信息的数据对象。

>[!warning]
>fp文件指针指向的位置**会随着对文件的读取和写入而改变**。
>
>1. 写入操作时位置变化
当你使用 fprintf(fp, "...", ...) 向文件写入数据时，文件指针会从当前位置开始，逐字节写入数据，并在写入完成后**自动移动到刚写入数据的末尾**。例如：``fprintf(fp, "%c%0%d.1.10", c, d);``。
>
>写入完成后，fp 指向的是写入内容的最后一个字节之后的位置。
>
>2. 读取操作时位置变化
>当你使用 fscanf(fp, "...", &var) 或 fgets(s, size, fp) 从文件读取数据时，文件指针会从当前位置开始读取指定数量的数据，并在读取完成后**自动移动到已读数据的末尾**。例如：``fscanf(fp, "%d", &e);``和``fgets(s, 4, fp);``。
>
>每次读取后，fp 都会前**进到下一个可读位置**。

## 05 指向标准文件的指针
stdio.h 头文件把 3 个文件指针与三个标准文件相关联，C程序会自动打开这 3 个标准文件。

|标准文件|文件指针|通常使用的设备|
|:---:|:---:|:---:|
|标准输入|stdin|键盘|
|标准输出|stdout|显示器|
|标准错误|stderr|显示器|

这些都是指向FILE的指针。
