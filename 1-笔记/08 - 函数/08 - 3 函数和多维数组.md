将数组作为函数的**声明**或**形参**，实际上是声明了**指向某一地址的指针变量**和**将数组的地址传入函数**。
因此，函数可以**直接通过地址修改原数组中的值**。

具体声明方法见[09 - 3 指针和数组](/1-笔记/09%20-%20指针/09%20-%203%20指针和数组.md)

这里着重介绍函数与多维数组
## 01 多维数组作一维处理
```c
#include <stdio.h>
int sum(int *, int );
int main()
{
    int num[3][4] = {{1, 2, 3, 4}, {2, 3, 4, 5}, {3, 4, 5, 6}};
    int i, j;
    int total = 0;
    for(i=0; i<3; i++){
        total += sum(num[i], 4);
    }
    printf("%d", total);
    return 0;
}
int sum(int num[], int m)
{
    int result = 0;
    int i;
    for(i=0; i<m; i++){
        result += num[i];
    }
    return result;
}
```
其中，``total += sum(num[i], 4);``这一句中，num[i]就是**一维数组首元素的地址**，因此函数原型和函数定义中声明的是**一维数组的形参**。

## 02 多维数组作形参
```c
#include <stdio.h>
int sum(int (*)[4]);
int main()
{
    int num[3][4] = {{1, 2, 3, 4}, {2, 3, 4, 5}, {3, 4, 5, 6}};
    int total = 0;
    total += sum(num); //对应传入二维地址。
    printf("%d", total);
    return 0;
}
int sum(int (* num)[4])
{
    int result = 0;
    int i, j;
    for(i=0; i<3; i++){
        for(j=0; j<4; j++){
            result += num[i][j];
        }
    }
    return result;
}
```
其中，``int sum(int (*)[4]);``函数原型也可以写成``int sum(int [][4]);``

>[!warning]
>注意⚠️：函数原型和函数定义中，``int sum(int num[][]);``这种声明是错误的。在编译中，编译器会**将数组表示法转换为指针表示法**，如果第二个括号为空，编译器就不知道如何处理。

>[!important]
>一般而言，声明一个指向N维数组的指针时，只能且必须省略第一个括号里的值。

## 有关变长数组(VLA)
C99中新增了变长数组，允许使用变量作为**数组的长度和数组的维度**。如下：
```c
int num1 = 3;
int num2 = 4;
int num[num1][num2];
```

>[!warning]
>注意⚠️：语句``int num[num1][num2]={0}``可能会出现**兼容性问题**，有的编译器允许这样做，而有的编译器要求**不能再声明中初始化它们**。

>[!important]
>变长数组有一些限制，变长数组必须是**自动存储类别**。这意味着无论**在函数中声明还是作为函数形参声明**，都**不能**使用static或extern存储类别说明符，而且，**不能在声明中初始化它们**。最终，C11把变长数组作为一个可选特性，而不是必须强制实现的特性。
